[h1]Building Themes 2[/h1]

Apply building themes to districts and cities. Only buildings in the selected themes will grow in a district. Subscribe to theme mods from the workshop, or create your own themes in-game.

[b]Building Themes 2[/b] is a community-maintained fork of the original [b]Building Themes[/b] by [url=https://github.com/boformer]boformer (Sebastian Schöner)[/url], updated to run on modern Cities: Skylines with Harmony 2.x.

[i]⚠️ Maintenance notice: This fork is maintained on a best-effort basis. Updates may be infrequent. Critical bugs will be fixed; new features will be added when time allows. Please check GitHub for known issues before reporting.[/i]

[url=https://github.com/roberto-naharro/BuildingThemes2/issues]► Report a bug on GitHub[/url]

[h1]Required[/h1]

[url=https://steamcommunity.com/sharedfiles/filedetails/?id=2040656402][b]► Harmony (Mod Dependency) — subscribe and enable this first[/b][/url]

[h1]Compatibility[/h1]

Compatible with [b]Cities: Skylines 1.18.x[/b] (tested on 1.18.1-f3).

[b]Not compatible with:[/b]
[list]
[*][b]81 Tiles (original)[/b] — use [url=https://steamcommunity.com/sharedfiles/filedetails/?id=2862121823]81 Tiles 2[/url] instead (fully supported)
[*][b]Building Themes (original)[/b] — unsubscribe the original before enabling this fork

Save games made with the original Building Themes are [b]fully compatible[/b] — district assignments load normally.

[h1]Quick Start[/h1]

[olist]
[*]Open a [b]district policy panel[/b] → go to the [b]Themes[/b] tab
[*]Click [b]Theme Manager[/b] → select a theme and include buildings in it (or import a style)
[*]Back in the [b]Themes[/b] tab, check [b]Enable Theme Management for this district[/b]
[*]Enable one or more themes using the checkboxes
[*]Zone the area — only buildings in the active themes will grow
[/olist]

If no theme is active, any growable building can spawn (vanilla behavior).

[h1]Buildings Are Not Spawning?[/h1]

Work through this checklist in order:

[b]1. Theme is empty or has no Level 1 buildings[/b]
Open the Theme Manager → select your theme → set the filter to [b]Included[/b]. If the list is empty, no buildings are in your theme. Every theme [u]must[/u] include at least one Level 1 building for a zone type (e.g. a Level 1 Low-Density Residential building). Higher levels require a Level 1 first.

[b]2. Workshop assets are not loaded[/b]
Workshop buildings you subscribed to but are not active in your current playset will show as [b]missing[/b] in the Theme Manager (red text). The mod can handle this three ways — see [i]Missing Assets[/i] below. Use [b]Workshop Dependencies[/b] (button in Theme Manager) to see exactly which assets are missing and copy their IDs for resubscription.

[b]3. Wrong zone type[/b]
The building's zone type must match the zone you placed. A commercial building will never spawn in a residential zone regardless of theme settings.

[b]4. Theme is not enabled for the district[/b]
Open the district policy panel → Themes tab → confirm [b]Enable Theme Management[/b] is checked and your theme has a checkmark next to it.

[b]5. Run Spawn Diagnostics[/b]
Open the district policy panel → Themes tab → [b]District Options[/b] → [b]Spawn Diagnostics[/b]. This shows how many buildings were accepted or rejected during the last theme compile, and lists any missing assets by name.

[h1]Missing Assets[/h1]

When a workshop building in your theme is not loaded (e.g. you unsubscribed or disabled it in Skyve), the mod can respond in three ways. Set the default in [b]Mod Options → Building Themes[/b], or override it per district in [b]District Options[/b].

[list]
[*][b]Skip[/b] — the missing building is quietly dropped. The theme still applies using only what is loaded. Areas may be sparse if many assets are missing.
[*][b]Fill with vanilla[/b] (default) — missing slots are supplemented with vanilla buildings of the same zone type and size. Your loaded theme buildings still appear; vanilla fills the gaps.
[*][b]Fall back to vanilla[/b] — if any building is missing in a size/level slot, that entire slot uses vanilla buildings only. The most conservative option: no sparse areas, but theme coverage is reduced.
[/list]

[h1]District Options[/h1]

Open via the [b]District Options[/b] button in the Themes tab. Per-district controls:

[list]
[*][b]Allow buildings not in any theme[/b] — turns themes into a blacklist: anything not explicitly excluded can spawn. Useful when you want to allow most buildings but block specific ones.
[*][b]Level behavior[/b] — what happens when a building tries to level up but your theme has no building for that level: vanilla fallback, cascade from lower level, or strict freeze.
[*][b]Missing asset handling[/b] — override the global missing asset mode for this specific district.
[*][b]Spawn Diagnostics[/b] — see accepted/rejected building counts and a list of missing assets for this district.
[/list]

[h1]Theme Manager[/h1]

Open from the [b]Themes[/b] tab. The left panel shows all themes; the middle panel shows buildings.

[b]Filters:[/b] Filter by zone type, level, size, origin (vanilla / workshop), asset status (loaded / missing / DLC locked), or name/Steam ID. The counter above the building list shows how many are spawnable in the current view.

[b]Quick actions:[/b]
[list]
[*][b]Include All / None / Valid[/b] — bulk-include or exclude buildings in the current filtered view
[*][b]Exclude Missing[/b] — remove all unloaded assets from the current filtered view in one click
[*][b]Workshop Dependencies[/b] — lists all workshop assets in the selected theme by status (loaded / missing), with a button to copy missing IDs to clipboard
[/list]

[b]Per-building options:[/b]
[list]
[*][b]Spawn rate[/b] — controls how often a building appears relative to others with the same type, level, and footprint
[*][b]Upgrade building[/b] — forces a specific building when this one levels up
[/list]

[b]How spawning works:[/b] A new zone always spawns a Level 1 building first — include one in your theme. It upgrades when the zone's requirements are met. The Level 2 building must have the same width and equal or smaller length than the Level 1. Example: a 2×3 L1 Low Residential can upgrade to a 2×3, 2×2, or 2×1 L2.

If a building is shorter than the plot, this mod shrinks the plot to the building's actual footprint (unless props or trees occupy the extra space). This allows tightly packed layouts like the UK Terraced Housing theme.

[h1]Asset Cloning[/h1]

The Theme Manager lets you clone buildings and assign different wealth levels — useful when you don't have enough assets for every level, or to add variety.

To clone: select a building → [b]Clone Building[/b] → enter a name and choose the wealth level. The clone appears in the building list.

[u]Important:[/u] Clones are generated at level load time. Return to the main menu and reload after creating clones.

[u]Important:[/u] If you delete a theme that contains clones, or disable the mod, those buildings disappear on next load. The save file itself is not broken.

Cloning can be disabled in mod options.

[h1]Mod Compatibility Tools[/h1]

For broader mod compatibility diagnostics — conflicts, disabled assets, missing dependencies — use [b]Skyve[/b]. If Skyve is installed, the [b]Spawn Diagnostics[/b] panel will note which missing assets may be disabled in your Skyve playset, and give you the IDs to re-enable them.

[h1]How The Mod Saves Your Data[/h1]
[list]
[*]Custom themes and options are stored in [b]BuildingThemes.xml[/b] in your Cities: Skylines install folder.
[*]District theme assignments are stored inside each save game (key: [i]BuildingThemes-SaveData[/i]).
[/list]

You do not need to edit these files — everything is configurable in-game.
