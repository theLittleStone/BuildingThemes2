[h1]Building Themes 2[/h1]

Apply building themes to districts — only buildings in the selected themes will grow in a zone. Subscribe to theme mods from the Workshop, or create and manage your own themes entirely in-game.

[b]Building Themes 2[/b] is a community-maintained fork of the original [b]Building Themes[/b] by [url=https://github.com/boformer]boformer[/url], updated for Cities: Skylines 1.21.1-f9 with Harmony 2.x.

[b]► Full documentation, troubleshooting & changelog:[/b]
[url=https://github.com/roberto-naharro/BuildingThemes2]github.com/roberto-naharro/BuildingThemes2[/url]

[h1]Required[/h1]

[url=https://steamcommunity.com/sharedfiles/filedetails/?id=2040656402][b]► Harmony (Mod Dependency) — subscribe and enable this first[/b][/url]

[h1]Key Features[/h1]

[list]
[*][b]Per-district themes[/b] — assign different building sets to each district
[*][b]Theme Manager[/b] — browse and filter by zone, level, size, height, origin, DLC/CCP, or name; create, rename, and delete themes
[*][b]Bulk actions[/b] — Include All / None / Valid (loaded + correct dimensions) / Exclude Missing in the current filtered view
[*][b]Per-building options[/b] — spawn weight, forced upgrade target, plop, bulldoze all instances
[*][b]Workshop Dependencies[/b] — see missing assets, copy IDs, or subscribe in one click
[*][b]Spawn Diagnostics[/b] — accepted/rejected counts, missing-asset list, and live non-theme building count per district
[*][b]Missing asset handling[/b] — Skip / Fill with vanilla / Fall back to vanilla (global default + per-district override)
[*][b]Level behavior[/b] — Vanilla fallback (default) or Strict freeze when a theme has no buildings for the next level
[*][b]Auto-bulldoze[/b] — gradually replace non-theme buildings in a district with themed ones; optional sub-option also removes buildings that don't match the district's active specialization (farming, eco, tourism…) so a district fully transitions to its new specialization; configurable pace (Gentle / Normal / Aggressive)
[*][b]Cluster against existing buildings[/b] — force new buildings to spawn next to existing ones and slide them along the road so their walls touch, eliminating the small gaps that arise when a building's mesh is narrower than its lot allocation; ideal for wall-to-wall row-house looks but works with any theme
[*][b]Asset Cloning[/b] — duplicate a building and assign a different wealth level (experimental)
[*][b]Spawn weight[/b] — control how often a building appears relative to same-type peers
[*][b]Size preference[/b] — bias spawning toward Biggest / Widest / Deepest / Smallest / Tallest / Shortest / Random footprints per zone type, with Gentle / Moderate / Strong / Absolute strength
[*][b]Blacklist mode[/b] — invert the theme: everything spawns except what you explicitly exclude
[*][b]Import from District Styles Plus[/b] — convert DSP styles into BT2 themes with one click (see below)
[/list]

[h1]District Options[/h1]

Each district has its own options panel (Themes tab → District Options):
[list]
[*]Enable/disable theme management per district
[*]Blacklist mode and auto-bulldoze toggles
[*]Level behavior and missing asset handling overrides
[*]Size preference — one dropdown per zone type (Residential / Commercial / Industrial / Office) plus preference strength
[*]Spawn Diagnostics button
[/list]

[h1]District Styles Plus[/h1]

If you have [url=https://steamcommunity.com/sharedfiles/filedetails/?id=1204126181]District Styles Plus[/url] styles, import them into BT2 with one click:
[list]
[*]Load a city that has your DSP styles defined
[*]Open [b]Mod Options → Building Themes → District Styles Plus[/b]
[*]Click [b]Import from District Styles Plus[/b]
[/list]

Each imported style becomes a theme named [b][DSP] style name[/b]. Only growable buildings are included. Already-imported styles are skipped on repeat runs. Use [b]Rename Theme[/b] in the Theme Manager to clean up the prefix afterwards.

[b]Compatibility note:[/b] Do not enable BT2 theme management and assign a DSP style to the [b]same district[/b] — the two mods filter the same spawning call and will conflict. Use one system per district. The import feature is the recommended migration path.

DSP does not need to be active for the import — its style packages are loaded by the game whenever a city is open.

[h1]Compatibility[/h1]

Compatible with [b]Cities: Skylines 1.21.1-f9[/b] and later 1.x. Save games from the original Building Themes are [b]fully compatible[/b].

[b]Not compatible with:[/b]
[list]
[*][b]81 Tiles (original)[/b] — use [url=https://steamcommunity.com/sharedfiles/filedetails/?id=2862121823]81 Tiles 2[/url] instead
[*][b]Building Themes (original)[/b] — unsubscribe the original before enabling this fork
[*][b]Building Simulation Overhaul[/b]
[*][b]Runways and Taxiways[/b] — use [url=http://steamcommunity.com/sharedfiles/filedetails/?id=465127441]Airport Roads[/url] instead
[/list]
